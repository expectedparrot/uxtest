# UXR Study Type Examples

This directory organizes reusable `uxtest` examples by common UXR jobs to be
done. Each subfolder contains:

- a Markdown guide explaining when and how to run that study type
- an EDSL `AgentList` persona export pattern
- a fixture template and run command
- `log.html` inspection guidance
- notes about how to interpret technical and narrative reports
- human screenshot validation guidance
- follow-on study suggestions

Available study types:

- [Task Discovery](task_discovery/README.md): first-impression orientation,
  first clicks, misunderstanding, and hesitation before action.
- [Conversion Path Testing](conversion_path_testing/README.md): whether
  visitors can reach demo, signup, pricing, checkout, contact, or another target
  action.
- [Information Architecture](information_architecture/README.md): whether
  visitors can find expected content such as docs, security, pricing,
  integrations, support, or examples.
- [Enterprise Buying Research](enterprise_buying_research/README.md): whether
  enterprise stakeholders can find enough product, trust, technical, risk, and
  commercial evidence to continue evaluation.
- [Competitive Benchmark Studies](competitive_benchmark_studies/README.md):
  same-task comparison across sites, competitors, variants, or before/after
  designs.
- [Content Comprehension](content_comprehension/README.md): whether visitors
  understand messaging, audience, value proposition, claims, jargon, and next
  steps.
- [Feature Findability](feature_findability/README.md): whether visitors can
  determine if a product supports a feature, integration, workflow, or use case.
- [Onboarding Activation](onboarding_activation/README.md): whether newly
  signed-up or invited users reach a first meaningful product action.
- [Post-Login Workflow Testing](post_login_workflow_testing/README.md): whether
  authenticated users can complete role-specific product workflows from a known
  state.
- [Accessibility Inclusive UX](accessibility_inclusive_ux/README.md): whether
  tasks work for mobile-only, low-confidence, plain-language, low-vision,
  keyboard-oriented, or unfamiliar-domain visitors.
- [Longitudinal Regression](longitudinal_regression/README.md): repeat known UX
  tasks over time to track fixes, regressions, and expected flaws.
